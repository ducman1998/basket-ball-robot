This document summarizes how the controller maps between robot motion and wheels (low-level), and how high-level handlers generate motion commands from pose errors.

== Low-Level Control
`MainboardController` receives `TwistStamped` and forwards `vx`, `vy`, `wz` to:

[source,python]
----
self.controller_kin.move(
    x_speed=vx,
    y_speed=vy,
    rot_speed=wz,
    thrower_speed_percent=thrower_percent,
    servo1=servo_speed,
)
----

Inside `OmniMotionRobot.move`, the robot velocity vector is:

`v = [wz, vx, vy]^T`

and wheel command vector is computed by Jacobian mapping:

`u = K * H * v`

- `H`: 3x3 omni-wheel Jacobian (built from wheel angles, wheel radius `r`, and center-to-wheel distance `d`).
- `K = gear_ratio * encoder_resolution / (2*pi*pid_control_freq)` converts wheel rad/s to mainboard ticks/cycle.

Code-equivalent (short):

[source,python]
----
v = np.array([rot_speed, x_speed, y_speed])
wheel_speeds = self.wheel_to_mb_unit * (self.jacobian @ v)
s1, s2, s3 = (int(round(ws)) for ws in wheel_speeds)
----

For the reverse direction (wheel -> robot velocity), odometry uses:

`v_robot = H^+ * w` (pseudo-inverse Jacobian, with `w` from wheel encoder increments).

== High-Level Control
Handlers (`BaseHandler`, `ManpulationHandler`) compute a pose error transform `T_error` in robot frame, then generate `(vx, vy, wz)`.
The goal position for manipulation is extracted from the Image Processor node, which provides ball positions directly in the robot base frame `(x_robot, y_robot)`.

=== Twist Error (SE(3) log map)

In manipulation position control, translation is first converted mm -> m, then:

`xi_error = se3ToVec(MatrixLog6(T_error))`

and the controller uses:

- `vx_error = xi_error[3]`
- `vy_error = xi_error[4]`
- `wz_error = xi_error[2]`

Code-equivalent:

[source,python]
----
xe_log = mr.MatrixLog6(x_la_target)
xe_vec = mr.se3ToVec(xe_log)
vx_error = xe_vec[3]
vy_error = xe_vec[4]
wz_error = xe_vec[2]
----

=== PID Law

For each channel (`vx`, `vy`, `wz`) the discrete PID form is:

`u[k] = Kp*e[k] + Kd*(e[k]-e[k-1])*f + Ki*sum(e)/f`

where `f = maneuver_rate`.

Code-equivalent (from `compute_pid`):

[source,python]
----
output = kp * error
output += kd * (error - prev_error) * maneuver_rate
output += ki * cumm_error
prev_error = error
cumm_error += error / maneuver_rate
----

`BaseHandler` also has a simplified forward alignment controller using heading error:

`wz_error = atan2(r21, r11)`

`vx = Vmax * sin(-wz_error) * 1.25`, `vy = Vmax * cos(wz_error)`, and `wz` by PID on `wz_error`.
