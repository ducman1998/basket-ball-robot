== 🚀 Development Setup

This project uses https://pre-commit.com/[pre-commit] to enforce code style and quality
(Black, isort, Ruff, Mypy, and more).

=== Install dev dependencies
[source,shell]
----
pip install -e .[dev]
cd software && pip install -r requirements.txt
----

=== Install Git hooks
[source,shell]
----
pre-commit install
----

This sets up a Git pre-commit hook so checks run automatically on every commit.

=== (Optional) Run on all files once
[source,shell]
----
pre-commit run --all-files
----

== ROS2 Nodes Quick View
- `mainboard_controller_node`: receives `cmd_vel`, sends wheel/thrower/servo commands, publishes wheel feedback.
- `odometry_node`: converts wheel feedback to odometry (`/odom`) and TF.
- `image_processor_node`: detects balls/basket/markers and publishes `/image/info` + visualization image.
- `game_logic_controller_node`: high-level behavior state machine (search, align, grab, throw, recovery).
- `teleop`: manual keyboard control for testing.

== Running Nodes
Before running commands, source your ROS2 workspace and install this package.

=== Run Individual Nodes
[source,shell]
----
# teleop
ros2 run teleop start_teleop

# mainboard controller
ros2 run basket_robot_nodes start_mainboard_controller

# odometry
ros2 run basket_robot_nodes start_odometry

# image processor
ros2 run basket_robot_nodes start_image_processor

# game logic controller
ros2 run basket_robot_nodes start_game_logic_controller
----

=== Launch Core Functional Stack
Launch file: `software/ros_nodes/src/basket_robot_bringup/launch/robot.launch.py`

Starts:
- `mainboard_controller_node`
- `odometry_node`
- `image_processor_node`

[source,shell]
----
ros2 launch basket_robot_bringup robot.launch.py
----

Optional custom parameter file:
[source,shell]
----
ros2 launch basket_robot_bringup robot.launch.py \
	param_file:=/absolute/path/to/setting.yaml
----

=== Launch Game Logic Controller
Launch file: `software/ros_nodes/src/basket_robot_bringup/launch/game_logic.launch.py`

Starts:
- `game_logic_controller_node`

[source,shell]
----
ros2 launch basket_robot_bringup game_logic.launch.py
----

Optional custom parameter file:
[source,shell]
----
ros2 launch basket_robot_bringup game_logic.launch.py \
	param_file:=/absolute/path/to/setting.yaml
----

NOTE: `game_logic_controller_node` depends on perception and motion topics, so in normal operation run `robot.launch.py` first, then `game_logic.launch.py`.
